# Локальный кодинг-ассистент — пошаговая настройка

Цель: получить рабочего AI-ассистента прямо в VS Code, который работает
**локально на твоей видеокарте** (8 ГБ VRAM), без интернета и без API-ключей.

После настройки у тебя будет:

- **Chat с AI прямо в VS Code** (как Copilot Chat, только локально).
- **Inline-подсказки при наборе кода** (tab autocomplete).
- **`@codebase` поиск** — задаёшь вопрос, модель видит весь твой проект.
- **Модель Qwen2.5-Coder 7B** — лучший open-source кодер для этого размера.

Всё железо: твой GPU 8 ГБ + Ollama + VS Code + Continue.dev.

Время настройки: **~30 минут один раз**. После этого работает всегда.

---

## Шаг 1. Установить Ollama (5 минут)

Ollama — это сервер, который запускает модель на твоём GPU и даёт к ней HTTP API.

1. Идёшь на [ollama.com/download/windows](https://ollama.com/download/windows)
2. Скачиваешь `OllamaSetup.exe`, запускаешь.
3. После установки Ollama запустится автоматически в трее (серая иконка с ламой).
4. Проверка — открой **PowerShell** и выполни:

   ```powershell
   ollama --version
   ```

   Должно напечатать версию (например `ollama version is 0.5.x`).

**Если командной строке не видно ollama:** перезапусти PowerShell. Если всё ещё нет — перезагрузи компьютер.

---

## Шаг 2. Скачать модели (10–15 минут, зависит от интернета)

Тебе нужны три модели. Открой PowerShell и выполни по очереди:

```powershell
# Основная chat-модель (~4.5 GB, будет отвечать на твои вопросы)
ollama pull qwen2.5-coder:7b

# Модель для tab-autocomplete (~1 GB, быстрые подсказки при наборе)
ollama pull qwen2.5-coder:1.5b-base

# Embedding-модель для @codebase поиска (~270 MB)
ollama pull nomic-embed-text
```

Пока качается — можешь идти к шагу 3.

Проверить что всё скачалось:

```powershell
ollama list
```

Должно показать все три модели.

### Почему именно эти модели

- **qwen2.5-coder:7b** — Qwen2.5-Coder от Alibaba, в квантизации Q4_K_M. На момент релиза (Q4 2024) это сильнейшая открытая модель 7B для кода: побеждает CodeLlama 13B, DeepSeek-Coder 6.7B и многих 70B общих моделей на HumanEval / MBPP. Занимает ~4.5 ГБ VRAM, работает на 8 ГБ комфортно.
- **qwen2.5-coder:1.5b-base** — быстрая модель для inline-подсказок. Большую 7B нельзя использовать для tab autocomplete — будет тормозить. 1.5B даёт подсказки за ~100 мс.
- **nomic-embed-text** — превращает код в векторы для поиска по `@codebase`. Работает на CPU, VRAM не съедает.

## Шаг 4. Установить Continue.dev в VS Code (2 минуты)

Continue.dev — это расширение, которое встраивает чат с AI прямо в VS Code
и ходит в Ollama за ответами.

1. Открой VS Code.
2. Иди в Extensions (Ctrl+Shift+X), найди **Continue**, установи (автор: Continue Dev, Inc).
3. После установки слева появится иконка Continue.

---

## Шаг 5. Подключить Continue к твоим моделям (1 минута)

В папке `local_assistant/` лежит готовый файл `continue_config.json` с
правильными настройками под твои 8 ГБ и скачанные модели.

**Скопируй его в конфиг Continue:**

Путь конфига Continue на Windows:

```
C:\Users\r1881\.continue\config.json
```

Что сделать:

1. Открой Проводник, вставь в адресную строку: `%USERPROFILE%\.continue`
2. Если папки нет — создай (или запусти Continue один раз, он создаст).
3. Замени (или создай) файл `config.json` содержимым из
   `local_assistant/continue_config.json`.

Можно одной командой в PowerShell:

```powershell
Copy-Item -Force `
  "C:\Users\r1881\Downloads\neural_kernel\local_assistant\continue_config.json" `
  "$env:USERPROFILE\.continue\config.json"
```

4. Перезапусти VS Code (или `Ctrl+Shift+P` → `Continue: Reload Window`).

## Шаг 6. Пользуешься (бесконечно)

### Чат

- Нажми иконку Continue слева или `Ctrl+L` — открывается chat.
- Напиши вопрос, например: «Как работает `TokenTransformerLM.generate`?»
- Ответ придёт от локальной 7B-модели.

### Контекст кода

В чате можно вставлять `@`-контексты:

- `@codebase какой класс отвечает за BPE tokenizer?` — модель найдёт по всему проекту.
- `@file какой-то_файл.py` — добавить конкретный файл.
- `@diff` — добавить текущий git diff.
- `@terminal` — последний вывод терминала.

### Inline-правки

Выдели блок кода → `Ctrl+I` → напиши «добавь обработку ошибок» / «оптимизируй» /
«перепиши на numpy». Модель предложит diff прямо в редакторе.

### Tab-autocomplete

Начни печатать код — появится серая подсказка, `Tab` принимает.
Это 1.5B модель, работает быстро.

## Troubleshooting

### Ollama не запускается / нет в PATH

```powershell
# Запустить вручную
& "$env:LOCALAPPDATA\Programs\Ollama\ollama.exe" serve
```

### Модель падает с OOM на 8 ГБ

Открой `config.json` в Continue, в секции `"models"` у `qwen2.5-coder:7b`
добавь:

```json
"completionOptions": {
    "numCtx": 2048
}
```

Это урежет контекст с 4096 до 2048 — сэкономит ~1 ГБ VRAM.

Если всё равно OOM — замени на `qwen2.5-coder:3b`:

```powershell
ollama pull qwen2.5-coder:3b
```

И в `config.json` меняешь `"model": "qwen2.5-coder:7b"` → `"qwen2.5-coder:3b"`.

### Continue не видит модели

- Проверь что Ollama запущена: в трее должна быть иконка ламы.
- Проверь что API отвечает: открой в браузере `http://localhost:11434`. Должно вывести `Ollama is running`.
- Перезагрузи VS Code.

### Tab-autocomplete тормозит

Уменьши `"maxTokens": 128` до `64` в секции `tabAutocompleteOptions`.

## Что потом с этим ассистентом
